# Backblaze B2

```yaml
backends:
  - id: b2
    name: "Backblaze B2"
    type: s3v4
    endpoint: https://s3.us-west-002.backblazeb2.com
    region: us-west-002
    access_key_env: B2_KEY_ID
    secret_key_env: B2_APPLICATION_KEY
    path_style: false
```

## Notes

- Use the **S3-compatible API key**, not the legacy B2 native API
  key. The S3 endpoint requires a key generated in the "Application
  Keys" section of the B2 console with S3-compatible scope.
- The endpoint base depends on the region of your bucket. Find it in
  the B2 console under **Endpoint**.
- `region` matches the URL slug (`us-west-002`, etc.) — B2 treats it
  as a literal label.
- B2 supports lifecycle, versioning, and bucket-level CORS but not
  full AWS-style bucket policy. The dashboard surfaces upstream errors
  for unsupported policy shapes.
